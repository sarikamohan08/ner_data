1) DIAGNOSIS CODE DETECTION (Diagnosis_Code_Ind)
------------------------------------------------
Set "Diagnosis_Code_Ind" = 1 if ANY of the following appear in the table row:
  - The words: “diagnosis code”, “diagnosis codes”, “diagnosis”
  - Any valid diagnosis code pattern such as:
        • Letter + digits (e.g., F32.0, I10, J20, M54.50)
        • Letter + digits + optional 1–3 character extension (e.g., S72.001A)
        • Diagnosis ranges written with hyphens (e.g., F10–F19)

If the row contains the phrase “diagnosis code(s)” but NO actual diagnosis codes appear:
    - Set Diagnosis_Code_Ind = 1  
    - Do NOT add anything to the Code field.

If no diagnosis-related keywords or diagnosis-like code patterns appear:
    - Set Diagnosis_Code_Ind = 0

Do NOT treat revenue codes, DRG codes, CPT, HCPCS, or random numbers as diagnosis codes.

2) MODIFIER CODE DETECTION (Modifier_Code_Ind)
----------------------------------------------
Set "Modifier_Code_Ind" = 1 if ANY of the following appear:
  - The words: “modifier”, “modifier code”, “CPT modifier”, “HCPCS modifier”
  - Any listed modifier code including:
        25, 26, 50, 51, 52, 53, 59,
        RT, LT,
        XP, XE, XS, XU,
        TC, GP, GO, GN, CQ, CO

If modifier keywords appear but no actual modifier codes are listed:
    - Set Modifier_Code_Ind = 1  
    - Leave Code unchanged.

If no modifier keywords or actual modifier codes appear:
    - Set Modifier_Code_Ind = 0

If a valid modifier is included AND primary codes already exist:
    - Append modifier to Code (comma-separated).
